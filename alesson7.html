Data Migration

Data Migration

  • is the process of transferring data between storage types, formats or computer systems. It is a key conconsolidation.
  • sideration for any system implementation, upgrade or Typically, data migration occurs during an upgrade of existing hardware or transfer to a completely new system.
  • occurs for a variety of reasons, including server or storage equipment replacements, maintenance or upgrades, application migration, website consolidation and data center relocation.

Reason for Data Migration

Categories of Data Migration

  • Data is stored on various media in files or databases, and is generated and consumed by software applications which in turn support business processes.
  1. Storage Migration
    • A business may choose to rationalize the physical media to take advantage of more efficient storage technologies. This will result in having to move physical blocks of data from one tape or disk to another, often using virtualization techniques.
  2. Database Migration
    • Similarly it may be necessary to move from one database vendor to another, or to upgrade the version of database software being used. The latter case is less likely to require a physical data migration, but this can happen with major upgrades.
  3. Application Migration
    • changing application vendor – for instance a new platform will inevitably involve substantial transformation as almost every application or suite operates on its own specific data model and also interacts with other applications any systems within the enterprise application integration environment.
  4. Business Process Migration
    • Business processes operate through a combination of human and application systems actions, often orchestrated by business process management tools. When these change they can require the movement of data from one store, database or application to another to reflect the changes to the organization and information about customers, products and operations.
    • Examples of such migration drivers are mergers and acquisitions, business optimization and reorganization to attack new markers or respond to competitive threat.

Stages of Data Migration

  • Pre Migration - Analysis, Mapping, Design
  • Migration - Transform, Normalize and backup
  • Post Migration - Quality Control, Clean-up, Maintenance.

Pre-Migration

  • Analysis - The analysis phase of data migration should be scheduled to occur concurrently with the analysis phase of the core project. The aim of the analysis phase in data migration projects is to identify the data sources that must be transported into the new system. For example what is my source database and what is my target database.
  • Mapping - Mapped each source field to target field. Eg. In my source table I have emp_id, salary as column in target database what should be my column name and its datatype.
  • Design - After you have decided upon the legacy data sources and have conducted thorough data analysis, you must begin the roster selection. This involves going through the list of data elements from each and every source data structure, and deciding whether to migrate each one.

Migration

  • Transform - After the design phase, Transform phase starts. In this phase data transform happen from source to target system. For this organization can use different ETL(Extract, Transform, Load) process.
  • Normalizze - After moving the data to Target system, data normalization required. Such as remove inconsistence data, remove duplicate data, storing data based on Normal form. Also moving data from stage layer to Target layer.
  • Back Up - This is one of the important phase of the migration. After transforming data from source to target state organization need to create backup process. This will help during database maintenance and recovery process.

Post Migration

  • Quality Control- This is the import phase in Post migration process. Here testing team need to check target database table structure, column type and its value. Also need to map this value with source system value. If any error occur then raise an defect and assign to the concern team. Also ask Business to verify their system whether the output is as expected or not.
  • Clean-Up - During migration process data transformation happened through different channels, such as file system, Temporary tables etc. After the migration process complete need to clean these temporary system.
  • Maintenance - In all organization they have separate team which take care of this process, they maintain the Database, do housekeeping. Organization spend huge amount in the maintenance phase.

Disadvantages of Data Migration

  • Migration addresses the possible obsolescence of the data carrier, but does address the fact that certain technologies which run the data may be abandoned altogether, leaving migration useless.
  • Time consuming – migration is continual process, which must be repeated every time a medium reaches obsolescence, for all data objects stored on a certain media.
  • Costly – an institution must purchase additional data storage media at each migration.

Cloud Migration and Cloud Service Providers

  • Is the process of moving data, applications or other business elements from an organization’s onsite computers to the cloud, or moving them from one cloud environment to another.
  • The process of transitioning to a different cloud provider is known as cloud service migration
  • A cloud provider is a company that offers some component of cloud computing – typically infrastructure as a service, software as a service or platform as a service to other businesses or individuals. Cloud providers are sometimes referred to as cloud service providers

Examples of Cloud Service Providers

  • Microsoft Azure
  • IBM Cloud
  • Oracle Cloud
  • OpenStack
