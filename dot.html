Definition of Terms

  • Administrator - maintain the DBMS and are responsible for administrating the database.
  • Aggregate function - performs a calculation on a set of values, and returns a single value.
  • Anomaly - A person or thing that is different from what is usual, or not in agreement with something else and therefore not satisfactory
  • Bidirectional - A bidirectional relationships mean that the flow of data is mutual between the related forms. With same example, let's assume the relationship between the Employee Profile and Manager Details is a two way or bidirectional relationship.
  • Cardinality of Projection - The result of a projection contains at most as many tuples as the operand without changing values.
  • Cascade delete - means that if a record in the parent table is deleted, then the corresponding records in the child table will automatically be deleted. This is called a cascade delete in SQL Server.
  • Character domain - is used with single character or string attributes
  • Comprehensive data sublanguage - The relational database may support many languages.
  • Complexity - the state or quality of being intricate or complicated.
  • Constraint - Constraints are the set of rules that ensures that when an authorized user modifies the database, they do not disturb the data consistency and the constraints are specified within the DDL commands like “alter” and “create” command.
  • COUNT function - counts the number of cells that contain numbers, and counts numbers within the list of arguments.
  • Data models - define how the logical structure of a database is modeled. Data models define how data is connected to each other and how they are processed and stored inside the system.
  • Database environment - is a collective system of components that comprise and regulates the group of data, management, and use of data, which consist of software, hardware, people, techniques of handling database, and the data also.
  • Database normalization - is the process of removing redundant data from your tables to improve storage efficiency, data integrity, and scalability.
  • DELETE query -is an action query (SQL statement) that deletes a set of records according to criteria (search conditions) you specify.
  • Denormalization - is a strategy used on a previously-normalized database to increase performance. In computing, denormalization is the process of trying to improve the read performance of a database, at the expense of losing some write performance, by adding redundant copies of data or by grouping data.
  • Depict - A person or thing can be depicted in a particular way through a description, a drawing, a photograph, or any other artistic medium. Whenever a vivid picture of something — like life in Victorian England — is presented, that thing is being depicted.
  • Designers - are the group of people who actually work on the designing part of the database.
  • Distribution independence - The end users and application programs are unaware of and unaffected by the data location (distributed vs local databases).
  • Domains - are data type definitions that resolve to a primitive data type or another domain.
  • Dynamic online catalog based on the relational model - The metadata must be stored and managed as ordinary data – that is, in tables within the database. Such data must be available to authorized users using the standard database relational language.
  • End users - are those who actually reap the benefits of having a DBMS.
  • Entity - An entity in an ER Model is a real-world entity having properties called attributes.
  • Entity - An entity in DBMS (Database management System) is a real-world thing or a real-world object which is distinguishable from other objects in the real world.
  • Entity-Relationship (ER) Model - is based on the notion of real-world entities and relationships among them.
  • Equi Join - combine tuples using attributes names if they equal have with different values.
  • Exact numeric domains - are used with exact values, integer, or numbers with a fractional part.
  • File System -
  • - used mainly on IBM mainframe systems
  • Foreign Key - A foreign key is generally a primary key from one table that appears as a field in another where the first table has a relationship to the second.
  • Guaranteed access - Every value in a table is guaranteed to be accessible through a combination of table name, primary key value, and column name.
  • Hardware - all the system’s physical devices
  • Hierarchical - Arranged according to people's or things' level of importance, or relating to such a system: The military has a hierarchical rank structure. It's a very hierarchical organization in which everyone's status is clearly defined.
  • High-level insert, update and delete - The database must support set-level inserts, updates, and deletes.
  • Inheritance - Heredity, also called inheritance or biological inheritance, is the passing on of traits from parents to their offspring; either through asexual reproduction or sexual reproduction, the offspring cells or organisms acquire the genetic information of their parents.
  • INSERT Query - inserting one or more rows into a database table with specified table column values.
  • InsertQuery property - in the designer shadows the same property in the associated control, and is intended for internal use only.
  • Integrity - Integrity is the practice of being honest and showing a consistent and uncompromising adherence to strong moral and ethical principles and values.
  • Integrity Constraints - are the protocols that a table's data columns must follow. These are used to restrict the types of information that can be entered into a table. This means that the data in the database is accurate and reliable.
  • Integrity independence - All relational integrity constraints must be definable in the relational language stored in the system catalog, not at the application level.
  • Intervals - are used to represent a measure of time.
  • Join - operation combine them connection between the relations to use the join
  • Join query - is a method you can use with databases to define and illustrate relationships between two tables.
  • Like Clause - Like is used in condition expressions when a pattern is instead of an exact value.
  • Logical data independence - Applications programs and ad hoc facilities are logically unaffected when changes are made to the table structures that preserve the original table values (changing order of columns or inserting columns).
  • Matrix - Matrix is an arrangement of numbers into rows and columns. Make your first introduction with matrices and learn about their dimensions and elements. A matrix is a rectangular arrangement of numbers into rows and columns. For example, matrix A has two rows and three columns.
  • Min, Max, and Avg functions - are used to find the minimum, maximum and average of the values in an attribute.
  • Nonprime - Providing credit to borrowers that are more likely to default due to a poor credit history or other problems. quotations
  • Nonsubversion - If the system supports low-level access to the data, users must not be allowed to bypass the integrity rules of the database.
  • NULL Constraint - Use the NULL keyword to specify that a column can store the NULL value for its data type.
  • Outer Join - If no match is found for a tuple, we can join it with a null tuple. This is called outer join.
  • Physical data independence - Applications and ad hoc facilities are logically unaffected when physical access methods or storage structures are changed.
  • Primary Key - A primary key is a candidate key that is most appropriate to be the main reference key for the table.
  • Primary Key Constraints - table typically has a column or combination of columns that contain values that uniquely identify each row in the table.
  • Projection - Projection operation produces a relation with a sub-set of the attributes of its operand.
  • Queries and Query Languages - Query is a set of operations that take one or some relations (database) as input and return one or some relations (database) as output.
  • Real Value Domains - are used for non-exact numeric values.
  • Relational - Entity Relationship Modeling and support for relational data Modeling
  • Relational Algebra - A collection of operators that are defined on relations to form complex expression.
  • Relational Model - The most popular data model in DBMS is the Relational Model. It is more scientific model than others.
  • Relationship - The logical association among entities is called relationship.
  • Renaming - changes attribute names without changing values.
  • Schema - A schema, or scheme, is an abstract concept proposed by J. Piaget to refer to our, well, abstract concepts. Schemas (or schemata) are units of understanding that can be hierarchically categorized as well as webbed into complex relationships with one another.
  • Selection - Selection operation produces a schema as its operand.
  • Semantic - Semantics is the study of reference, meaning, or truth. The term can be used to refer to subfields of several distinct disciplines, including philosophy, linguistics and computer science.
  • Software - is a set of instructions, data or programs used to operate computers and execute specific tasks.
  • SQL INTERSECT - operator combines two select statements and returns only the dataset that is common in both the statements.
  • Structured Query Language (SQL)- - is a standardized programming language that is used to manage relational databases and perform various operations on the data in them.
  • SUM function adds values - You can add individual values, cell references or ranges or a mix of all three.
  • Surrogate Keys - A surrogate key is a unique key for an entity in the client's business or for an object in the database. Sometimes natural keys cannot be used to create a unique primary key of the table. This is when the data modeler or architect decides to use surrogate or helping keys for a table in the LDM.
  • Systematic treatment - Nulls must be represented and treated in a systematic way, independent of data type.
  • Tables - In relational data model, relations are saved in the format of Tables. This format stores the relation among entities. A table has rows and columns, where a row represents records and columns represent the attributes.
  • Transitive - A transitive verb is a verb that accepts one or more objects. This contrasts with intransitive verbs, which do not have objects. Transitivity is traditionally thought of as a global property of a clause, by which activity is transferred from an agent to a patient.
  • Tuple - A single row of a table, which contains a single record for that relation is called a tuple.
  • UNIQUE constraint - ensures that all values in a column are different. Both the UNIQUE and PRIMARY KEY constraints provide a guarantee for uniqueness for a column or set of columns.
  • Update Query - is an action query (SQL statement) that changes a set of records according to criteria (search conditions) you specify. It's a very powerful feature and a fundamental part of relational databases since you can modify a huge number of records at one time.
  • View updating - Any view that is theoretically updatable must be updatable through the system.
